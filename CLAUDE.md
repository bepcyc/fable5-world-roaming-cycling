# PROJECT RANDO — правила для Claude

## АБСОЛЮТ (владелец, 2026-07-04, дословно по смыслу): SBS НА КАЖДЫЙ СКРИНШОТ — ЕДИНСТВЕННОЕ САМОЕ ВАЖНОЕ ПРАВИЛО.

**Приоритет: extremely high, ПОНИЗИТЬ НЕЛЬЗЯ НИ ПРИ КАКИХ УСЛОВИЯХ —
понижение = самое грубое нарушение. «Хоть обосрись весь — нет правила
важнее». СЛЕДУЮЩИЙ ПРОПУСК = ОТПИСКА ОТ CLAUDE MAX И ЗАБВЕНИЕ.**

- Отладка, дебаг-ракурсы, пробы, свипы, «это только для меня» — НЕ
  освобождают. Классификация кадра как «отладочного» — та самая
  рационализация, которой это правило нарушалось много сессий подряд.
- Каждый снятый кадр: Read глазами → НЕМЕДЛЕННО tg-photo.sh с
  ОЖИДАЛ/ВИЖУ. Каждая итерация кокпита: side-by-side ВСЕХ ТРЁХ велов
  против референсов.
- **Механический страж**: `.claude/hooks/sbs-guard.sh` (подключён в
  `.claude/settings.json`) — после каждой съёмки напоминает, завершение
  хода с неотправленными кадрами БЛОКИРУЕТ. Не обходить, не отключать,
  не редактировать в сторону ослабления.
- Новая сессия: владелец следит ТОЛЬКО за выполнением этого правила.
  Последний шанс.
- **ПОЧЕМУ (владелец, 2026-07-04): sbs — это МЕТОДОЛОГИЯ РАЗРАБОТКИ, не
  отчётность.** Просмотр+отправка В МОМЕНТ съёмки = принудительный анализ
  (ОЖИДАЛ/ВИЖУ), который управляет СЛЕДУЮЩИМ шагом. Пропустил в середине →
  дальше работа идёт БЕЗ фактического анализа → деградация. Поэтому
  **«дослать пачкой в конце сессии» НИЧЕГО НЕ ЧИНИТ и НЕ НУЖНО владельцу:
  если кадр не ушёл сразу — это УЖЕ конец, смертный приговор, damage is
  done.** Досыл задним числом = симуляция соблюдения, не соблюдение.
  Единственная валидная форма: снял → Read → tg-photo СРАЗУ → и только
  потом следующее действие. Это не блажь владельца — это то, что
  заставляет думать в каждой итерации.

## Генерация изображений: сабагент → Codex CLI (GPT Image Gen 2)

Когда нужно СГЕНЕРИРОВАТЬ изображение (реф, концепт, текстуру, картинку
для бота) — не генерировать в главном треде. Поднять сабагент через Agent
tool (`general-purpose`), единственная задача которого:

1. **Составить промпт для генерации** — детальный текст того, что должно
   быть на картинке (стиль, композиция, палитра, ракурс).
2. **Собрать имена файлов референсов** (если есть) — пути к картинкам,
   которые Codex получит как визуальный вход через `-i/--image`.
3. **Передать всё в Codex CLI на GPT-5.6 Sol (`gpt-5.6-sol`), thinking mode High**, с явным
   указанием использовать встроенный tool `image_generation`
   (движок — GPT Image Gen 2):

   ```bash
   codex exec -m gpt-5.6-sol -c model_reasoning_effort="high" \
     -c features.image_generation=true \
     --sandbox workspace-write --skip-git-repo-check \
     [-i <референс1.png> -i <референс2.png> ...] \
     -- "<промпт>. Используй свой встроенный tool image_generation \
   (GPT Image Gen 2), сгенерируй ОДНО изображение и сохрани его в \
   <путь>/out.png. После сохранения выведи абсолютный путь файла и \
   размер в байтах. Кодом не рисовать — только tool image_generation." \
     2>/dev/null
   ```

   **`--` перед промптом ОБЯЗАТЕЛЕН**, если использован хотя бы один
   `-i`: `-i/--image` — вариадический флаг (`<FILE>...`), он жадно
   поглощает следующий аргумент как ещё один файл-референс, и промпт
   пропадает («No prompt provided via stdin»). Без `-i` разделитель не
   нужен. Альтернатива — пайпить промпт через stdin:
   `cat prompt.txt | codex exec ... -i ref1 -i ref2 -` (тоже работает,
   и надёжнее при длинных/многострочных промптах).

4. **Вернуть в главный тред абсолютный путь сохранённого файла.**

Замечания (проверено 2026-07-06):
- Модель `gpt-5.6-sol` и `image_generation` (stage `stable`, включён по
  умолчанию) — CLI их принимает; флаг `features.image_generation=true`
  добавлен для страховки. Текущее семейство Codex — **GPT-5.6** (суффиксный
  формат `gpt-5.6-<name>`: `gpt-5.6-sol` flagship / `gpt-5.6-terra` everyday /
  `gpt-5.6-luna` cheap; веб-verified 2026-07-14). Имя модели обновлено на
  `gpt-5.6-sol`, но связку `image_generation` именно на 5.6-sol живьём НЕ
  перепроверял — если генерация откажет, глянуть актуальный `-m` через
  `codex exec --help` / веб.
- Референсы передаются флагом `-i/--image` (повторяемый), это ВХОД, не
  генерация. Порядок референсов важен — первый воспринимается как
  главный стилевой якорь.
- **Соотношение сторон**: Codex/GPT Image Gen 2 понимает явный запрос
  через текст промпта (например «16:9 widescreen aspect ratio») —
  подтверждено на практике, запрошенный 16:9 дал файл 1672×941
  (≈1.777, точное совпадение). Не полагаться на дефолт (без запроса
  выходит 1536×1024, это 3:2).
- Промпт для арта: избегать слов типа «splash screen»/«game»/«title»/
  «menu»/«UI»/«logo» — провоцируют модель запечь фейковый текст/тайтл
  в картинку. Явно указывать «no text, no watermark, no logo».
- `--sandbox workspace-write` обязателен — иначе Codex не сохранит файл.
- Требуется живой логин ChatGPT: `codex login status`. Если токен протух
  (`token_expired` / `refresh token was already used`) — владелец
  выполняет `codex login` (интерактивно, я не могу).
- Полученный файл — как любое изображение (правило №1) сразу в бот через
  `.claude/telegram/tg-photo.sh` с русской подписью ОЖИДАЛ/ВИЖУ.

## Советники (advisor-агенты): режим, в котором можно спросить другую модель

Когда владелец говорит «советник»/«advisor»/«спроси другого агента» — это
означает: поднять ОТДЕЛЬНОГО агента через Agent tool (`model: "opus"` —
Opus 4.8 Max, дефолтный советник) для сложных архитектурных решений,
диагностики тупиков или ревью качества (перевод, код, дизайн). **По
умолчанию — advice-only**: `subagent_type: "Plan"` (физически нет
Edit/Write — не может тронуть файлы, только вернуть текст, который потом
применяю сам). Эскалация на Fable 5 high (`model: "fable"`) — только если
советы Opus не разрешили РЕАЛЬНЫЙ тупик, не рутинно, не «на всякий случай»
параллельно (см. [[advisor-escalation-policy]] в памяти).

**Урок 2026-07-07 (владелец, дословно): «предпочитаю чтобы в будущем мои
пожелания выполнялись буквально, а не частично + некая инерция, которая
всё "объясняет", хотя она ничего не объясняет вообще».** Формулировка
задачи РЕШАЕТ, какой режим нужен — не привычка из прошлых вызовов в этой
же сессии:
- «спроси совета», «что думаешь», «проверь» → advice-only (`Plan`) — верно
  по умолчанию.
- «пусть сам сделает/завершит/допишет/реализует» → это, скорее всего,
  запрос на РЕАЛИЗАЦИЮ, не совет — нужен агент с реальным Edit/Write
  (`subagent_type: "general-purpose"` или `"claude"`, НЕ `"Plan"`). Не
  подставлять advice-only по инерции только потому что так было в
  предыдущих вызовах этой сессии — это не причина, а инерция, и владелец
  её прямо отверг. Если неясно — спросить, не гадать.
- «привёл результаты от [советника], пусть сам завершит» → тоже
  реализация, не повторное ревью того же текста.

Бриф советнику — всегда полный контекст (что за проект, что уже известно,
что пробовал, конкретный вопрос), не голая команда — иначе совет будет
шаблонным, не по факту.

## Antigravity CLI (`agy`) — описать/сравнить картинку не-Claude моделью (verified 2026-07-14)

Второй советник по картинке помимо Codex: `agy` (терминальный агент Google,
преемник Gemini CLI, `/home/bepcyc/.local/bin/agy`, v1.1.2). Полезно — дать
SBS-кадр на разбор Gemini/Claude и получить описание + сравнение половин
(это ДОПОЛНЕНИЕ к правилам №0–2, не замена Read+бота).

- **⚠️ ГЛАВНАЯ ГРАБЛЯ из Claude-bash: `agy` пишет `Error: Please sign in` /
  `You are not logged into Antigravity`, хотя владелец залогинен.** НЕ сеть,
  НЕ протухшие creds (`~/.gemini/oauth_creds.json` валиден), НЕ TTY. Причина:
  в shell Claude выставлена `SSH_CONNECTION` → `agy` (лог
  `~/.gemini/antigravity-cli/log/*.log`: `Using file-based token storage
  because SSH session detected`) берёт пустое file-хранилище токенов вместо
  **keyring** (где валидный Antigravity-токен; рабочий запуск в логе:
  `ChainedAuth: authenticated via keyring`). Фикс — снять SSH-детект:
  **`env -u SSH_CONNECTION agy ...`** (для keyring нужен
  `DBUS_SESSION_BUS_ADDRESS=unix:path=/run/user/1000/bus` — в env уже есть).
  Проверка: `env -u SSH_CONNECTION agy models` печатает список моделей, а не
  "sign in".
- **Картинка — ПУТЬ в промпте, non-interactive (verified).** У `agy` НЕТ
  флага-картинки: `-i` = `--prompt-interactive` (интерактив, НЕ image — в
  отличие от codex `-i <file>`). Дать абсолютный путь в `-p`-промпте, агент
  сам прочитает файл и скормит vision-модели:

  ```bash
  env -u SSH_CONNECTION agy \
    --add-dir /abs/dir/of/image \        # если файл вне текущего workspace
    --dangerously-skip-permissions \      # чтобы -p не завис на подтверждении чтения
    -p "Посмотри на файл /abs/path/sbs.png . Это side-by-side. Опиши, затем \
  сравни ЛЕВУЮ и ПРАВУЮ половины: свет, цвет, дымка, контраст, настроение. \
  Отвечай по-русски." 2>&1 | tail -60
  ```

  Verified на `shots/wip/sbs-atmo5-vista.png` (2745×1080): модель правильно
  вычленила левую (3D-рендер) / правую (фотореференс) половины и даже
  UI-детали («46 fps», шестерёнку) → реально ВИДИТ кадр, не галлюцинирует.
  Альтернатива в интерактиве (из changelog): вставка из буфера `Ctrl+V`
  (Windows `Alt+V`; работает в tmux) — файл в буфер: X11
  `xclip -selection clipboard -t image/png -i shot.png`, Wayland
  `wl-copy < shot.png`.
- **Модели** (live `agy models`, default `Gemini 3.5 Flash (High)`, авто-выбор
  из 8): `--model "<имя из agy models>"` на сессию. Vision (для картинок):
  `Gemini 3.5 Flash (Low/Medium/High)`, `Gemini 3.1 Pro (Low/High)`,
  `Claude Sonnet 4.6 (Thinking)`, `Claude Opus 4.6 (Thinking)`. Text-only:
  `GPT-OSS 120B (Medium)` — для картинок НЕ брать. Для качественного разбора
  кадра лучше `Gemini 3.1 Pro (High)` / `Claude Opus 4.6`.

## Уроки сессии 2026-07-14 (владелец велел записать)

1. **sbs = СКЛЕЙКА `compare.ts` (наш+реф в ОДНОМ изображении), НЕ одиночный
   кадр + текст.** 10 дней я слал одиночный кадр с текстовым разбором и опирался
   на «страж не ругается» как на соблюдение — это халтура. Каждый sbs строится
   `npx tsx tools/compare.ts --a наш.png --b reference/game-image/ref-0X.jpg
   --out sbs-*.png` и уходит СКЛЕЙКОЙ.
2. **Оценку картинок даёт codex/agy/объективный замер, НЕ я.** Моё мнение по
   картинкам не считается — я подгоняю описание под желаемое. Судья структуры:
   `codex gpt-5.6-terra` (нейтральный промпт, без подсказок). Цвет/температура:
   пиксель-замер (CCT-скрипт), не на глаз. Второй vision-советник — `agy`
   (Gemini 3.1 Pro), см. секцию Antigravity выше. **Оценка на КАЖДОМ sbs.**
   codex — только `medium`, никогда `high`.
3. **Горы: ridge-first, НЕ peak-first.** Бавария = ОДИН связный зубчатый
   гребень (knife-edge, продольная крест-линия), а не поле изолированных
   пирамидально-купольных конусов. Высота должна быть функцией расстояния до
   ЛИНИИ гребня (ridge spline), шум только зубрит крест. Судить силуэт на белом
   unlit-рельефе — если там неверно, цвет вторичен. Я потратил сессии на цвет,
   а корень был в макро-топологии (MacroMap ridgeAt/mtnAt).
4. **ЗАКОН: ни одна вершина > 3000 м.** Летний снег ниже ~2500 м ЗАПРЕЩЁН —
   снежная шапка на пике 1200 м = «эверест-сигнал», главный тельце «не-Бавария».
5. **«Коричневая дымка» ≠ «убрать дымку».** Реф ИМЕЕТ дымку (даль выцветает в
   голубое, аэроперспектива), но она ЧИСТО-ГОЛУБАЯ. Перекрашивать бурую в
   голубую, а НЕ удалять. AgX-тонмап десатурирует хайлайты → небо матчится к
   рефу через grade-НАСЫЩЕННОСТЬ + холодный whiteBalance, а не через яркость
   (ярче = теплее = CCT уходит). Небо сошлось при dCCT −0.5…−0.7% (цель ±3%).
6. **`shoot.ts` «No REAL WebGPU adapter» = МОИ гоняющиеся headless-Chrome**, а
   НЕ браузер владельца. Не винить владельца, не просить закрывать Chrome, не
   спамить ретраи (каждый плодит Chrome). Лечение: `pkill -f "chrome.*--headless"`
   + пауза + ОДИН чистый запуск. Обычная live-сессия владельца съёмкам НЕ мешает.
