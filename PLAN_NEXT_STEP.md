# WAR PLAN — офлайн Android APK (вариант B) + Linux binary

> Боевой план перехода этого движка (three.js **WebGPU + TSL-compute +
> Web Bluetooth**, ~30k LOC TS, Vite) в две упаковки: **Linux-бинарь**
> (Electron, уже собран) и **самодостаточный офлайн Android APK** (вариант B:
> нативная оболочка + бандл `dist/` + loopback-сервер + Chrome Custom Tab).
>
> Формат — военный: у каждого рубежа есть **риск → как обнаружим → куда
> сворачиваем**. Плюс три стратегических разворота, меняющих само решение.
> Задача плана — чтобы при первом же «пошло не так» не гадать, а знать манёвр.

---

## 0. Конечное состояние (что считаем победой)

- `just linux-binary` → `dist-electron/*.AppImage` + `linux-unpacked/laas`,
  запускается на этой машине с аппаратным WebGPU. **[СДЕЛАНО]**
- `just android-apk` → `android/app/build/outputs/apk/**/*.apk`, ставится на
  телефон, работает **офлайн**, рендерит мир на WebGPU через установленный
  Chrome, читает BLE-датчики. **[В РАБОТЕ — остановлено по приказу «не меняй»]**
- Оба пути задокументированы (`docs/PACKAGING.md`) и воспроизводимы.

## Железный контекст (разведданные сессии — на них строится весь план)

Это не мнения — проверено web-ресёрчем + Codex в этой сессии. Ломается план,
если ломается один из этих фактов:

1. **WebGPU на Android живёт ТОЛЬКО в Chrome-браузере (и Chrome-движковых
   поверхностях: Custom Tabs / TWA). В Android System WebView WebGPU по
   умолчанию НЕТ (2025-2026).** `androidx.webkit` не имеет переключателя.
   → значит Capacitor/Tauri/голый WebView (бандлят офлайн, но на System
   WebView) **не дают WebGPU**, а WebGL2-фолбэка у движка нет (compute-шейдеры).
2. **`http://127.0.0.1` — secure context** (потенциально доверенный origin) →
   WebGPU и Web Bluetooth там разрешены.
3. **Custom Tab = движок Chrome пользователя** → любая web-фича Chrome доступна.
   Пинить на `com.android.chrome`.
4. **Immersive-фуллскрин** в Custom Tab не гарантирован: есть
   `setUrlBarHidingEnabled(true)` + Fullscreen API (`navigationUI:"hide"`),
   но kiosk-режима нет. Полностью без UI — только TWA (а он требует хост).
5. **Android 16 «Advanced Protection Mode» отключает WebGPU в Chrome.**
6. Требования устройства: **Chrome ≥121, Android 12+**, GPU Adreno/Mali
   (Qualcomm/ARM). PowerVR — только Chrome ≥139 / Android 16.
7. **BLE не переписывать**: страница в Chrome → Web Bluetooth работает,
   нативный плагин и Bluetooth-permission хосту НЕ нужны.
8. Свой Chromium в APK (убрать зависимость от Chrome) = проект-эмбеддинг
   браузера на **месяцы**, не задача-обёртка.

## 1. Ось наступления (happy path) — 6 рубежей

```
Рубеж 0  Тулчейн:   Android SDK + Gradle + JDK21
Рубеж 1  Web build: vite (ANDROID_APK=1, base '/') → dist/
Рубеж 2  Gate:      BrowserGate пускает hostname 127.0.0.1
Рубеж 3  Нативка:   AssetHttpServer(127.0.0.1:47862) + MainActivity(CustomTab→chrome)
Рубеж 4  Сборка:    dist→assets/www, gradle assembleDebug → APK
Рубеж 5  Устройство: install, офлайн, WebGPU on, BLE, playable
```

Рубежи 0-4 — «в тылу» (сборка, проверяемо здесь). **Рубеж 5 — туман войны**:
единственное, что нельзя доказать без реального телефона.

## 2. Рубеж 0 — Тулчейн

**Цель**: Android SDK (build-tools 34, platform-34, platform-tools), Gradle 8.7,
JDK 21.

| # | Риск | Как обнаружим | Манёвр |
|---|------|---------------|--------|
| 0.1 | `JAVA_HOME` → Java 8 (sdkman `current`) ломает sdkmanager/gradle | стектрейс `sun.misc.Launcher` / ClassLoader | **[УЖЕ СЛУЧИЛОСЬ, РЕШЕНО]** форсить `JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64` во ВСЕХ Android-командах и в `just android-apk` |
| 0.2 | Gradle не достаёт `google()`/`mavenCentral` (offline/proxy) | resolve-таймаут при первом build | заранее прогреть `~/.gradle` онлайн; далее `--offline`; крайний случай — vendored оффлайн-репо |
| 0.3 | AGP ↔ Gradle ↔ JDK рассинхрон | build падает на версии | пин-матрица: **AGP 8.5.2 / Gradle 8.7 / Kotlin 1.9.24 / JDK 21 / compileSdk 34**. Менять только всю матрицу целиком |
| 0.4 | sdkmanager лицензии не приняты | `> Accept licenses` | `yes \| sdkmanager --licenses` (уже в бутстрапе) |

**Статус**: SDK установлен, Gradle 8.7 скачан, JDK-21 путь известен. Рубеж взят.

## 3. Рубеж 1 — Web build под APK

**Цель**: `dist/` с root-абсолютными путями (для раздачи с корня loopback).

| # | Риск | Как обнаружим | Манёвр |
|---|------|---------------|--------|
| 1.1 | base `/laas/` вместо `/` → 404 всех ассетов, белый экран | `curl http://127.0.0.1:PORT/` → пустой `#app`; DevTools 404 | `ANDROID_APK=1` → base `/` (как `ELECTRON=1`) |
| 1.2 | Абсолютные `/splash/**`, `/rider/**` в index.html/CSS | preload/ассеты 404 | сервер раздаёт `assets/www` с корня → пути живут. Если нет — переезд на относительные / хеш-манифест |
| 1.3 | Воркеры (`RoadGridWorker`) не резолвятся из file/loopback | воркер не стартует, дороги без сетки | vite бандлит воркер как ассет; проверить путь на loopback |
| 1.4 | Большой JS-чанк (1.35 МБ) грузится медленно с сервера на телефоне | долгий boot | приемлемо (локально); при нужде — сжатие/сплит |

## 4. Рубеж 2 — BrowserGate

**Цель**: не отвергать мобильный Chrome, но не ослаблять WebGPU-гейт.

| # | Риск | Как обнаружим | Манёвр |
|---|------|---------------|--------|
| 2.1 | `isMobileDevice()` → `failLoud` до пробы WebGPU (`BrowserGate.ts:65`) | на телефоне — экран «recommend a computer» | пускать при `location.hostname === "127.0.0.1"` (это заведомо наша APK-обёртка) |
| 2.2 | Обход гейта сломает реальную диагностику на неподдерживаемом устройстве | пользователь видит краш вместо внятного сообщения | **оставить** проверку `navigator.gpu` (строка 86) — при отсутствии WebGPU показать причину (Chrome<121 / APM / GPU) |
| 2.3 | правка гейта заденет owner-логику | typecheck / ревью | правка минимальна (одно условие), не трогать остальную ветку |

## 5. Рубеж 3 — Нативная оболочка

**Цель**: `AssetHttpServer` (только `127.0.0.1`, порт 47862) + `MainActivity`
(Custom Tab, пин на `com.android.chrome`).

| # | Риск | Как обнаружим | Манёвр |
|---|------|---------------|--------|
| 3.1 | Порт 47862 занят → `BindException` | краш на старте | fallback на эфемерный порт. **Цена**: origin меняется → IndexedDB/localStorage/BLE-гранты сбрасываются. Принять |
| 3.2 | Гонка: Custom Tab открылся раньше, чем сервер слушает | «connection refused» в табе | старт сервера → дождаться `isAlive` → потом `launchUrl` |
| 3.3 | Chrome не установлен / нет `com.android.chrome` | `PackageManager` null | сообщение + ссылка в Play; крайний случай — дефолтный CCT-провайдер (с риском отсутствия WebGPU) |
| 3.4 | Раздача не с `127.0.0.1`, а с `0.0.0.0` | сервер виден в LAN — дыра | биндить строго `127.0.0.1` (secure-context + не наружу) |
| 3.5 | Неверные MIME (`.wasm`,`.glb`,`.webmanifest`) → модуль не грузится | MIME-ошибка в консоли | таблица MIME в сервере (как в `electron/main.cjs`) |
| 3.6 | Сервер живёт после сворачивания приложения | утечка сокета/батарея | `onDestroy` → `server.stop()`; при желании — foreground-lifecycle |

## 7. Рубеж 5 — Устройство (ТУМАН ВОЙНЫ)

Единственный рубеж, недоказуемый без телефона. Здесь заготовлены развязки.

| # | Риск | Как обнаружим | Манёвр / разворот |
|---|------|---------------|-------------------|
| 5.1 | `navigator.gpu` undefined в Custom Tab на устройстве | probe-страница / `chrome://gpu` | диагностировать причину ↓ и вести пользователя |
| 5.1a | Chrome < 121 | версия Chrome | попросить обновить Chrome |
| 5.1b | Android < 12 | `Build.VERSION` | устройство не поддерживается (сообщить честно) |
| 5.1c | **Android 16 Advanced Protection Mode** глушит WebGPU | APM включён | инструкция выключить APM |
| 5.1d | GPU не в списке (PowerVR pre-139) | `chrome://gpu` | требовать Chrome ≥139 |
| 5.2 | WebGPU есть, но краш/TDR/OOM на мобильном GPU | GPU-процесс падает, чёрный экран | `?preset=low&dpr=1` + `limitcap=mobile` (пол лимитов уже есть); дальше — резать фичи |
| 5.3 | FPS неиграбельный | замер на устройстве | ниже пресет / меньше мир / меньше растительности; крайний — принять как «демо» |
| 5.4 | BLE `requestDevice` не работает в Custom Tab | чузер не всплывает | проверить secure-context + user-gesture. Если Custom Tab режет BLE-чузер → **ПОВОРОТ к TWA** или нативный BLE-плагин |
| 5.5 | Тулбар Custom Tab не прячется приемлемо | визуально | принять тонкий тулбар, либо **ПОВОРОТ к TWA** (иммерсив ценой офлайна) |

## 8. Стратегические развороты (меняют САМО решение)

Если тактические манёвры рубежа 5 не спасают — переходим на другую доктрину.
Порядок = от дешёвого к дорогому.

### ПОВОРОТ 1 — Hosted TWA (жертвуем офлайном ради иммерсива/надёжности)
- **Триггер**: Custom Tab не даёт приемлемый WebGPU/BLE/фуллскрин на целевых
  устройствах, ИЛИ владельцу критичен полностью-без-UI экран.
- **Что делаем**: `just android-apk` в TWA-режиме (Bubblewrap), деплой `dist/`
  на HTTPS (GitHub Pages / домен). Immersive, но нужен публичный хост, не офлайн.
- **Цена**: вечный хостинг + первая загрузка онлайн (дальше service worker
  может кешировать → «офлайн после первого запуска» — половинчато).
- **Что переиспользуем**: тот же `dist/`, тот же гейт-обход, манифест уже готов.

### ПОВОРОТ 2 — Нативный рантайм (убираем зависимость от Chrome)
- **Триггер**: требование запускаться БЕЗ установленного Chrome, или Chrome-
  зависимость признана неприемлемой.
- **Что делаем**: нативный движок (Jetpack WebGPU Kotlin-биндинги) —
  переписать scene-graph/TSL/compute/рендер. **Месяцы.**
- **Цена**: фактически второй проект. Только если стратегически обязательно.

### ПОВОРОТ 3 — Заморозить Android, отгрузить десктоп
- **Триггер**: WebGPU-на-мобильном широко незрел под наши нужды (compute/лимиты).
- **Что делаем**: релизим **Linux-бинарь (Electron, готов)** + Windows/macOS тем
  же Electron; Android ждёт созревания мобильного WebGPU. Возврат к B позже.
- **Цена**: нет Android сейчас. Дёшево, честно, обратимо.

## 9. Точки невозврата и как их держать открытыми

- **`dist/` — общий актив** всех трёх Android-доктрин (B / TWA) и десктопа.
  Не завязывать его на loopback-специфику жёстко: base переключается флагом
  (`ANDROID_APK`/`ELECTRON`), не хардкодом. → переключение доктрины дёшево.
- **BrowserGate**: обход через `hostname===127.0.0.1` работает и для B; для TWA
  нужен `?apk=1`/`?nogate=1`. Держать оба.
- **BLE за интерфейсом `Transport`** уже абстрагирован — если ПОВОРОТ потребует
  нативный BLE, это единственный шов, куда бить. Не размазывать BLE по коду.

## 10. Откат (rollback) — всё аддитивно и обратимо

Ничего из владельческого WIP не тронуто. Отмена перехода:
```
rm -rf android/ dist-electron/ electron/ docs/PACKAGING.md
git checkout -- Justfile vite.config.ts index.html .gitignore
# public/manifest.webmanifest + public/icons/icon-*.png — новые файлы, удалить при желании
npm remove electron electron-builder
```
Установленные SDK/Gradle — в `~/Android/Sdk` и `~/.cache/laas-gradle`, вне репо,
удаляются отдельно, репозиторий не засоряют.

## 11. Ворота проверки (gate per rubеж — «взято/не взято»)

- Р0: `sdkmanager --list` показывает build-tools;34, platform;34; `gradle -v` = 8.7/JDK21.
- Р1: `python -m http.server` над `dist/` + открыть в Chrome desktop с
  `--enable-unsafe-webgpu` → мир рендерится (base '/' корректен).
- Р2: тот же прогон с мобильным UA → гейт не блокирует, WebGPU-проба зелёная.
- Р3: `AssetHttpServer` юнит-смоук: `curl 127.0.0.1:47862/index.html` = 200,
  правильный MIME для `.wasm/.glb`.
- Р4: `apk/**/app-debug.apk` существует, `aapt dump badging` = верный пакет/minSdk.
- Р5: `adb install` + ручной прогон на телефоне; **каждый кадр — в бот с
  ОЖИДАЛ/ВИЖУ** (правило sbs действует и здесь).

### Резюме одной фразой
Наступаем по оси B (офлайн Custom-Tab APK). Рубежи 0-4 доказуемы здесь и почти
взяты; рубеж 5 (устройство) — туман, но на каждый его провал есть тактический
манёвр, а на исчерпание манёвров — три заготовленных стратегических разворота
(TWA → натив → заморозка), причём общий актив `dist/` делает любой разворот
дешёвым.
